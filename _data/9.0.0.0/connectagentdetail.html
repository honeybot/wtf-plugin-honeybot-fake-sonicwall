SMA Connect Agent Install and Usage

Introduction

The SMA Connect Agent replaces the browser plug-ins to launch the native applications. It can launch NetExtender, Virtual Assist, Virtual Meeting, Windows Remote Desktop Application and Citrix Remote Connection. Additionally, EPC and PDA on the login page depend on the SMA Connect Agent.

You can download the SMA Connect agent from:

https://[Hostname]/cgi-bin/clientdownloads

Install Steps

  1. Double click SMAConnectAgent.msi
  2. If User Access Control (UAC) pops up, click Yes to approve the installer to do the action
  3. If necessary, input the Administrator password to install the Connect Agent.

GUI Usage

Tray

The connect agent will add an icon to the system tray if you run it manually.

  • To access Proxy Settings, double-click the tray icon.
  • To access the menu, right click the tray icon

Proxy Settings

There are four options for proxy. Choose the appropriate one for your needs. Ask your administrator if you're unsure.

Log

The log will be opened by the Notepad, you can check the detail of the Connect Agent.

Install Steps

  1. Double click SMAConnectAgent.dmg to open the install window
  2. Drag SMAConnectAgent to the Applications folder

Important: The Connect Agent only runs under the Applications folder.

GUI Usage

Tray

The connect agent will add an icon to the menu bar if you run it manually.

  • To access the menu, click the tray icon.

Proxy Settings

There are four options for proxy. Choose the appropriate one for your needs. Ask your administrator if you're unsure.

Log

The log will be opened in the Console app.

Browser Warning

The Connect Agent will be launched by some actions such as NetExtender Connection, RDP-Native Bookmark. Some browsers may show an alert to confirm you want to run the Connect Agent. Please click Allow/OK/Launch Application button to launch the Connect Agent.
